## Plan of Work

### Phase 1: Fix state.json / last_scan_time Bug

**File:** `bot_v2.py`  
**Problem:** `last_scan_time` is never written to state.json. The bot sets `last_full_scan` internally but never persists it. Any logic relying on `state["last_scan_time"]` is broken.

**Fix:** After the scan loop completes (after `print("ok")`), add:
```python
state["last_scan_time"] = last_full_scan
save_state(state)
```

**Verify:** Check `data/state.json` after next scan — `last_scan_time` should show recent timestamp.

### Phase 2: Fix Dallas side=NO / fill_price Bug

**File:** `bot_v2.py` around lines 1920-1940  
**Problem:** When `side=NO`, the bot calls `simulate_fill(..., direction="buy_yes", ...)` instead of `"sell_yes"`. This means:
- The fill simulation uses the WRONG side of the AMM
- The position is recorded with the WRONG direction
- Exit price calculations will be wrong

**Fix:** In the NO-side branch, pass `direction="sell_yes"` to `simulate_fill()`. Also verify the position dict records `side="NO"` and that closing/resolved PnL uses the correct formula for NO positions.

**Additional fix:** The `[BUY]` print log should say `[BUY YES]` or `[SELL YES]` depending on side, not always `[BUY]`.

### Phase 3: Fix [EDGE] vs [BUY] EV Display Discrepancy

**File:** `bot_v2.py`  
**Problem:** `[EDGE] Dallas | edge=+0.2977` but `[BUY] | EV -0.29`. The `edge` printed in [EDGE] is the raw EV before fees/slippage. The `EV` printed in [BUY] should be the post-fill EV. But the [BUY] line is printing `best_signal['ev']` which may be stale or the wrong field.

**Fix:** After fill simulation, recompute EV using `fill_price` and `p` from the position dict. Print the post-fill EV in the [BUY] line. Make it clear: `EV (pre-fill)` vs `EV (post-fill)`.

### Phase 4: Verify EV Calculation End-to-End

**File:** `bot_v2.py`  
**Problem:** Manual analysis shows Atlanta Apr 21 76-77F bucket: YES=$0.885, Gaussian P~0.240. This should produce positive EV for YES side (p > market price). Bot found nothing.

**Verification steps:**
1. Load Atlanta Apr 21 market file
2. Get forecast and sigma
3. Manually compute Gaussian P for each bucket
4. Compare to what bot SHOULD have traded
5. Find where in `should_trade()` or signal generation it was rejected

**Expected positive EV scenario (Atlanta Apr 21, forecast 75.8F, sigma 1.46):**
- 76-77F bucket: market YES=$0.885, Gaussian P(bucket) ~ 0.240
- YES_EV = 0.240 * (1-0.885) - 0.760 * 0.885 = 0.0276 - 0.6726 = -0.645 ... negative
- Actually for YES: p=0.24, price=0.885. EV = p*(1-price) - (1-p)*price = 0.24*0.115 - 0.76*0.885 = 0.0276 - 0.6726 = **-0.645** (very negative)
- NO side: price=0.115. EV = (1-p)*(1-no_price) - p*no_price = 0.76*0.885 - 0.24*0.115 = 0.6726 - 0.0276 = **+0.645**
- So NO on 76-77F should have +0.645 EV!

The bot should be looking at the NO side for overvalued YES markets. The `side` selection logic needs to be verified.

**Verify the NO-side signal generation:**
- For each bucket: compute YES_EV and NO_EV
- Select `side = "NO" if no_ev > yes_ev else "YES"`
- Trade the side with higher EV

```bash
cd ~/.openclaw/workspace/alter-bot-v1

python3 -c "
import json, math, os

# Load Atlanta Apr 21 market
with open('data/markets/atlanta_2026-04-21.json') as f:
    m = json.load(f)

forecast = 75.8  # from last snapshot
sigma = 1.46     # sigma_override for Atlanta

print('Atlanta Apr 21 forecast:', forecast, 'F, sigma:', sigma)
print()

# For each bucket compute Gaussian P and EV
outcomes = m.get('outcomes', [])
for o in outcomes:
    q = o.get('question','')
    import re
    nums = re.findall(r'\d+', q)
    if len(nums) < 2: continue
    lo, hi = int(nums[0]), int(nums[1])
    yes_mkt = float(o.get('price', 0))
    no_mkt = 1.0 - yes_mkt
    vol = float(o.get('volume', 0))
    if vol < 100: continue
    
    # Gaussian P for bucket
    mid = (lo + hi) / 2
    from scipy.stats import norm
    p = norm.cdf(mid, loc=forecast, scale=sigma) - norm.cdf(lo, loc=forecast, scale=sigma)
    
    # EV for YES and NO
    yes_ev = p * (1 - yes_mkt) - (1 - p) * yes_mkt
    no_ev = (1 - p) * (1 - no_mkt) - p * no_mkt
    
    if abs(yes_ev) > 0.01 or abs(no_ev) > 0.01:
        print(f'  {lo}-{hi}F: YES={yes_mkt:.3f} NO={no_mkt:.3f} | P={p:.3f} | YES_EV={yes_ev:+.3f} NO_EV={no_ev:+.3f} vol={vol:.0f}')
"
```
